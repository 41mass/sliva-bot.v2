# Telegram-бот магазина сухофруктов «Слива»

Бот написан на Python + aiogram 3.x. Он умеет:

- показывать каталог сухофруктов;
- собирать корзину и проверять остатки на складе;
- оформлять заказ с телефоном, ФИО, адресом, датой/временем доставки и комментарием;
- сохранять клиентов по Telegram ID;
- предлагать использовать прошлые ФИО, телефон и адрес при повторном заказе;
- показывать покупателю его историю заказов;
- делать тестовую оплату через кнопку-заглушку;
- после тестовой оплаты еще раз проверять склад и списывать товар;
- присылать админу уведомления о новых и оплаченных заказах;
- управлять товарами через админку: цена, приход на склад, добавление и удаление позиции;
- смотреть заказы компактным списком, открывать нужный заказ по кнопке, смотреть клиентов и статистику продаж.

## Структура проекта

```text
sliva_bot/
├── bot/
│   ├── handlers/
│   │   ├── admin.py      # админ-панель в Telegram
│   │   └── user.py       # сценарии покупателя
│   ├── config.py         # переменные окружения
│   ├── db.py             # SQLite база, товары, заказы, клиенты
│   ├── formatters.py     # красивый текст сообщений
│   ├── keyboards.py      # кнопки
│   ├── main.py           # запуск бота
│   └── states.py         # состояния диалогов
├── data/                 # тут появится база sliva.db
├── .env.example          # пример настроек
├── Procfile              # запуск на Railway
├── railway.json          # команда запуска Railway
└── requirements.txt      # зависимости
```

## Шаг 1. Создайте бота в Telegram

1. Откройте Telegram.
2. В поиске найдите `@BotFather`.
3. Нажмите `Start`.
4. Отправьте команду `/newbot`.
5. BotFather спросит название. Напишите, например: `Слива магазин`.
6. Потом он спросит username. Он должен заканчиваться на `bot`, например: `sliva_dry_fruits_bot`.
7. BotFather выдаст токен вида `123456789:AA...`.
8. Сохраните этот токен. Это значение для `BOT_TOKEN`.

Никому не показывайте токен. Кто знает токен, тот может управлять ботом.

## Шаг 2. Узнайте свой Telegram ID

Это нужно, чтобы бот присылал вам уведомления о заказах.

1. В Telegram найдите `@userinfobot`.
2. Нажмите `Start`.
3. Бот покажет ваш `Id`, например `123456789`.
4. Это значение для `ADMIN_NOTIFY_ID`.

## Шаг 3. Запуск на компьютере

### Установите Python

1. Зайдите на <https://www.python.org/downloads/>.
2. Скачайте Python 3.12 или новее.
3. При установке на Windows обязательно поставьте галочку `Add Python to PATH`.
4. Нажмите `Install Now`.

### Подготовьте проект

Откройте терминал в папке `sliva_bot` и выполните:

```bash
python -m venv .venv
```

Активируйте виртуальное окружение.

На Windows:

```bash
.venv\Scripts\activate
```

На macOS/Linux:

```bash
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

### Создайте файл настроек

Скопируйте `.env.example` в новый файл `.env`.

В `.env` вставьте свои значения:

```env
BOT_TOKEN=токен_от_BotFather
ADMIN_PASSWORD=4458
ADMIN_NOTIFY_ID=ваш_telegram_id
DATABASE_PATH=./data/sliva.db
```

### Запустите бота

```bash
python -m bot.main
```

Если ошибок нет, откройте своего бота в Telegram и нажмите `Start`.

## Как пользоваться админкой

1. Напишите боту `/admin`.
2. Бот попросит пароль.
3. Введите `4458`.
4. Появится меню:
   - `Админ: товары`
   - `Админ: заказы`
   - `Админ: клиенты`
   - `Админ: статистика`

Первый человек, который введет правильный пароль, добавится в список админов в базе. Потом бот будет узнавать его по Telegram ID.

## Как работает заказ

1. Покупатель нажимает `Каталог`.
2. Выбирает товар и вес.
3. Переходит в `Корзина`.
4. Нажимает `Оформить заказ`.
5. Вводит телефон, ФИО, адрес, дату/время доставки и комментарий.
   Если клиент уже заказывал раньше, бот предложит кнопку `Оставить прошлые данные`.
6. Бот создает заказ со статусом `создан`.
7. Покупатель нажимает `Оплатить заказ (тест)`.
8. Бот еще раз проверяет остатки.
9. Если товара хватает, заказ становится `оплачен`, а товар списывается со склада.
10. Админ может поставить статус `выдан`, `отменен` или `возврат`.

Статусы:

- `создан` - заказ создан, но еще не оплачен;
- `оплачен` - тестовая оплата прошла, товар списан;
- `выдан` - заказ отдали клиенту;
- `отменен` - заказ отменен;
- `возврат` - заказ был оплачен, но по итогу нужно оформить возврат.

## Где потом подключать настоящую оплату

Сейчас оплата - это кнопка-заглушка в файле:

```text
bot/handlers/user.py
```

Функция:

```python
@router.callback_query(F.data.startswith("pay:"))
async def pay(...)
```

Когда будете подключать настоящие платежи Telegram, ЮKassa, CloudPayments или другой сервис, эту функцию нужно заменить на:

1. создание платежа;
2. ожидание успешного платежного события;
3. вызов `db.mark_paid_and_write_off_stock(...)` только после реальной оплаты.

То есть склад уже списывается в правильном месте: после оплаты и повторной проверки остатков.

## Деплой на Railway

Railway меняет интерфейс со временем, но общий путь такой.

### Подготовьте GitHub

1. Зайдите на <https://github.com/>.
2. Зарегистрируйтесь или войдите.
3. Создайте новый репозиторий.
4. Загрузите папку `sliva_bot` в репозиторий.

### Создайте проект на Railway

1. Зайдите на <https://railway.com/>.
2. Войдите через GitHub.
3. Нажмите `New Project`.
4. Выберите `Deploy from GitHub repo`.
5. Выберите репозиторий с ботом.
6. Railway сам увидит Python-проект и команду запуска из `railway.json`.

### Добавьте переменные

В Railway откройте ваш сервис, затем раздел `Variables`.

Добавьте:

```env
BOT_TOKEN=токен_от_BotFather
ADMIN_PASSWORD=4458
ADMIN_NOTIFY_ID=ваш_telegram_id
DATABASE_PATH=/data/sliva.db
```

По официальной документации Railway переменные окружения добавляются в разделе Variables, а start command можно задать отдельно, если автозапуск не сработал.

### Добавьте постоянное хранилище

SQLite база - это файл. Если не подключить постоянное хранилище, база может пропасть при пересборке сервиса.

В Railway нужен Volume:

1. Откройте проект Railway.
2. На рабочем поле проекта нажмите правой кнопкой мыши или используйте меню создания.
3. Выберите `Volume`.
4. Подключите Volume к сервису бота.
5. Mount path укажите `/data`.
6. Убедитесь, что переменная `DATABASE_PATH` равна `/data/sliva.db`.

В официальной документации Railway Volumes описаны как способ хранить постоянные данные сервиса.

### Если Railway попросит Start Command

Укажите:

```bash
python -m bot.main
```

## Важные замечания

- Для маленького магазина SQLite нормально подходит.
- Если заказов станет много, лучше перейти на PostgreSQL.
- Telegram не всегда позволяет написать человеку первым, если он раньше не писал вам лично. В админке клиент показан как `@username`, если он есть, или кликабельный Telegram ID.
- Пароль `4458` лучше заменить перед реальным запуском.
- Не публикуйте `.env` в открытый GitHub-репозиторий.
